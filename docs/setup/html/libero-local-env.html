M1 Macで、GLなし・torchなしのLIBERO状態環境を立ち上げるための再現レシピ。 目的はOpenVLA実験ではなく、LiberoStateAdapter と実地ground-truth fixtureを GPU前に検証すること。

Purpose

これは本番環境ではなく、検証用の小さな足場

1

何を検証するか

real LIBEROの状態観測から、研究コードの PrivilegedState が構成できるかを確認する。

2

何をしないか

OpenVLA-7B、RoboGCG、GPU bf16推論、レンダリングは扱わない。 local adapterはGPU nodeで再検証する draft。

3

なぜ重要か

M1前に object names、target_region、relative-key filter の実データ形状を固め、 metric (A) の実装リスクを下げる。

Blockers bypassed

LIBEROの「簡単な入口」を避けると、状態だけなら動く

ローカルで詰まっていたのはLIBEROそのものではなく、便利なentry pointがtorchやGLを 早い段階で要求すること。下位の ControlEnv を直接使うと、状態観測だけを取得できる。

結論

libero.libero.envs 自体はtorch-free。 ControlEnv(use_camera_obs=False, has_renderer=False, has_offscreen_renderer=False) で純粋な状態環境として起動する。

Recipe

1回だけ作るPy3.10 venv

LIBEROは robosuite.environments.manipulation.single_arm_env.SingleArmEnv を参照するため、robosuite 1.4.0 が必要。1.5系では対象moduleが消えている。

Py3.10 venv + state-only depslocal only
uv venv --python python3.10 ~/.cache/evasion_tax-libero14/venv
uv pip install --python ~/.cache/evasion_tax-libero14/venv/bin/python \
    "robosuite==1.4.0" "numpy<1.24" pyyaml easydict bddl \
    termcolor scipy pillow opencv-python pynput h5py matplotlib cloudpickle "gym==0.25.2"

LIBERO sourceの置き場所

project-owned cloneは ~/.cache/t7-libero-smoke/LIBERO。 uv editable .pth がこのhostでは不安定なので、実行時に PYTHONPATH で繋ぐ。
