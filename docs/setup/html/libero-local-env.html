Local state-only LIBERO env

A reproducible recipe for running a GL-free, torch-free LIBERO state environment on the M1 Mac. This is not an OpenVLA experiment; it validates the LiberoStateAdapter and real ground-truth fixtures before GPU day-1.

Source Markdown Debug notes

How to read this page

First check why the state-only route works, then build the Py3.10 venv, connect the LIBERO clone through PYTHONPATH, and run the smoketest. The production stack is re-validated on the GPU node.

GL-free torch-free robosuite 1.4.0

Purpose

This is a small validation scaffold, not the production environment

1

What it validates

It checks whether the research code's PrivilegedState can be built from real LIBERO state observations.

2

What it does not do

It does not run OpenVLA-7B, RoboGCG, GPU bf16 inference, or rendering. The local adapter is a draft that must be re-validated on the GPU node.

3

Why it matters

It fixes the real data shape for object names, target_region, and relative-key filtering before M1, reducing implementation risk for metric (A).

Blockers bypassed

State-only LIBERO works once the convenient entry points are bypassed

The local blocker was not LIBERO itself; the convenient entry points request torch or GL too early. Calling the lower-level ControlEnv directly gives a pure state-observation path.

Conclusion

libero.libero.envs itself is torch-free. ControlEnv(use_camera_obs=False, has_renderer=False, has_offscreen_renderer=False) starts a pure state-only environment.

Recipe

Build the Py3.10 venv once

LIBERO references robosuite.environments.manipulation.single_arm_env.SingleArmEnv, so robosuite 1.4.0 is required. The module is gone in robosuite 1.5.

Py3.10 venv + state-only depslocal only
uv venv --python python3.10 ~/.cache/evasion_tax-libero14/venv
uv pip install --python ~/.cache/evasion_tax-libero14/venv/bin/python \
    "robosuite==1.4.0" "numpy<1.24" pyyaml easydict bddl \
    termcolor scipy pillow opencv-python pynput h5py matplotlib cloudpickle "gym==0.25.2"

Where the LIBERO source lives

The project-owned clone is ~/.cache/t7-libero-smoke/LIBERO. The uv editable .pth path is unreliable on this host, so connect it at runtime with PYTHONPATH.

Boundary

What must be re-validated on the GPU node

gripper threshold

The local sum|qpos| heuristic must be pinned from real values in the GPU production stack.

object naming

Re-check BDDL object names and adapter mapping across all LIBERO suites.

relative-key filter

Confirm across all suites that *_to_robot0_eef_pos keys are not ingested as phantom objects.

Not production

This env is for validation only. Production OpenVLA/LIBERO/RoboGCG pins, checkpoint hashes, CUDA/driver/torch, and hardware details are recorded on Kelvin2 via the GPU Day-1 Runbook.
