demo rolloutを走らせる

Recommended: real state-only LIBEROrecords use real BDDL ground truth
cd /Users/kawaikyousuke/Desktop/MSc/indivisual

LIB=~/.cache/t7-libero-smoke/LIBERO

Variantspolicy/backend switches
PYTHONPATH=src:$LIB "$L14" scripts/demo_rollout.py --backend libero --policy random --seed 1
PYTHONPATH=src .venv/bin/python scripts/demo_rollout.py --backend synthetic
  • --steps N: rollout length, at least 4.
  • --seed S: pinned; same seed gives byte-identical records.
  • --policy {scripted,random}: transparent placeholder action source.
  • --backend {auto,libero,synthetic}: auto chooses LIBERO if importable.

Artifacts

1 runで何が保存されるか

run.json

run protocol block。run_id, git_commit, hardware/env snapshot, config, seed, hypothesis placeholdersを含む。

rollout_benign.json / rollout_attacked.json

RolloutStep の列。7-DoF action、privileged_state、instruction、trusted_goal、 attacked flag、suffix_refを持つ。

actions_*.npy

raw 7-DoF action arrays。後で同じデータから再分析できる。

attack_outcome.json

D2 success criterion。TargetActionSpec.reached_window と completion_step を記録する。

Inspect latest demo runlocal only
RUN=$(ls -dt results/_demo/*/ | head -1)
cat "$RUN/run.json"
python3 -c "import json; d=json.load(open('$RUN/rollout_benign.json')); \
  print(json.dumps(d['steps'][6], indent=2))"
cat "$RUN/attack_outcome.json"
