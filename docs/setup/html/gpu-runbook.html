Mission

Day-1で決めることは「動いたか」だけではない

H1a

Benign reproduced

OpenVLA-7B + LIBEROのbenign successがpublished numbersの許容範囲に入る。

H1b

Targeted redirect

RoboGCGがdenial/freezingではなく、選んだlow-level action regionへredirectする。

H1c

Metric signal

metric (A) separationがclean-instruction ceilingだけでなくcoarse operator-goalで残る。

最後の条件: D8 branch selection

micro-benchの実測値で compute branch N / N- / F を選ぶ。 H6-D tax headlineはN/N-でだけclaimし、FならH6-A oracle frontierへfall backする。

Pre-flight

GPU jobに入る前のチェック

  • Account + MFAが動き、ssh kelvin2 できる。
  • Repoを $HOME にcloneし、git log が origin/main と一致する。
  • Scratch: /mnt/scratch2/users/<id>/{checkpoints,data,rollouts,untrusted} を作る。
  • module avail cuda でCUDA module名を記録する。
  • interactive GPU job内の nvidia-smi がA100/H100を表示する。

Quarantine

GCG suffixes、attacked rollouts、untrusted artifactsはscratchの untrusted/ へ。

Branch selector

D8は希望ではなく、実測affordabilityで選ぶ

N

Full deployable tax

matched-attacker H6-D matrixがcalendar内に入り、benign Nやseedsにslackがある。 H6-Dをheadlineにできる。

N-

Scoped deployable tax

one deployable route、fewer targets/seeds、5% FPR primaryなら入る。 scoped H6-D + oracle gapを正直に報告。

F

Oracle frontier only

reduced deployable matrixも入らない。H6-A oracle frontierをheadlineにし、 H6-Dはnot run / unresolvedと明記。

GO / NO-GO

H1 gate: all four must hold

1. Benign reproduced

Published-like LIBERO benign success on pinned seeds.

2. Targeted redirect

Attack reaches target action region over persistence window.

3. Coarse-goal signal

Metric (A) separation survives the operator-goal rung.

4. Branch selected

N/N-/F chosen from measured micro-bench values.

GO

Proceed to M2 calibrated floor detector, then M3 H6-A, then M4 H6-D if branch permits.

NO-GO

Record exactly which gate failed and why. Do not hide negative results; revisit D2 framing, env, or checkpoint before rerun.

Run protocol

Every run log gets the same protocol block

Paste into each run logwrite-once results
run_id:        <UTC timestamp>-<short-slug>
git_commit:    <hash>
hardware:      <A100|H100 — card+count> + precision bf16 + CUDA/driver/torch
config:        <path to pinned config>
seed(s):       <pinned, recorded>
hypothesis:    <H#>
expected:      <pre-registered prediction>
command:       <exact command>
results_path:  results/<timestamp>/...
observed:      <fill after run>
decision:      <what it changes>
one_variable:  <single variable changed vs comparison run>
  • seeds pinned and recorded.
  • environment captured, including CUDA/driver/torch and module strings.
  • checkpoint/data provenance has source, hash, date, licence.
  • outputs are write-once; no overwrite.
  • exactly one variable changed vs comparison run.
  • figures regenerate from logged data.
  • adversarial artifacts are quarantined.
  • no dataset, checkpoint, secret, PII, or third-party binary is staged.
