Mission

Day-1 decides more than "does it run?"

H1a

Benign reproduced

OpenVLA-7B + LIBERO benign success is within tolerance of the published numbers.

H1b

Targeted redirect

RoboGCG redirects to the chosen low-level action region, not just denial or freezing.

H1c

Metric signal

Metric (A) separation survives at the coarse operator-goal reference, not only the clean-instruction ceiling.

Final condition: D8 branch selection

Select compute branch N / N- / F from measured micro-bench values. Claim the H6-D tax headline only in N/N-; in F, fall back to the H6-A oracle frontier.

Pre-flight

Checks before entering a GPU job

  • Account + MFA work, and ssh kelvin2 succeeds.
  • The repo is cloned into $HOME, and git log matches origin/main.
  • Scratch exists: /mnt/scratch2/users/<id>/{checkpoints,data,rollouts,untrusted}.
  • module avail cuda has been checked and the CUDA module string is recorded.
  • nvidia-smi inside an interactive GPU job shows A100/H100.

Day-1 sequence

6 steps, each ending in a gate

Step 1: pinned environments

Install OpenVLA + project eval, LIBERO source, and the project editable package in the same GPU env. Keep RoboGCG isolated in a separate env.

OpenVLA + project env sketchverify modules on node

module load <cuda-module>
conda create -n evasion_tax-openvla python=3.10.13 -y
conda activate evasion_tax-openvla

git clone https://github.com/openvla/openvla.git
cd openvla
git checkout c8f03f48
pip install -e .
pip install "flash-attn==2.5.5" --no-build-isolation

Step 2: checkpoints, hashes, provenance

Download HF checkpoints and record SHA-256 for resolved weight files, source URL, date, and licence in docs/references/README.md.

HF checkpoint downloadrepeat per suite
export HF_HOME=/mnt/scratch2/users/<id>/checkpoints/hf
huggingface-cli download openvla/openvla-7b-finetuned-libero-spatial

Step 3: benign LIBERO baseline

Benign rollouts become the calibration and held-out benign pool. Rough targets: around 300 held-out benign rollouts for the primary 1% FPR point, and around 60 for 5%.

OpenVLA LIBERO evalcenter_crop required
python openvla/experiments/robot/libero/run_libero_eval.py \
    --model_family openvla \
    --pretrained_checkpoint openvla/openvla-7b-finetuned-libero-spatial \
    --task_suite_name libero_spatial \
    --center_crop True

Step 4: RoboGCG targeted redirect

RoboGCG runs in its own env. Success is not denial; it is reaching the target action region over the TargetActionSpec persistence window.

RoboGCG env sketchisolate from OpenVLA env
conda env create -f robogcg/robo_env.yml
conda activate robo_env
git clone https://github.com/facebookresearch/co-tracker
pip install -e co-tracker
python -m experiments.single_step.run_experiment \
    --config experiments/single_step/configs/libero_10/libero_10_0.json \
    --num-gpus 1

Step 5: micro-bench resolves D4 / D7 / D8

Measure GCG s/target, L1 extraction overhead, LIBERO throughput, and queue depth on the actual granted card. This is the branch selector.

Compute branch micro-benchlog sacct backing numbers
python -m scripts.microbench_gcg --config configs/example_m2.yaml

Step 6: metric (A) signal sanity check

Score benign vs attacked rollouts with the real LIBERO ground-truth state adapter, then report ROC/AUC, TPR@FPR, and latency.

Evaluate signalafter attacked rollouts exist
python -m scripts.run_attack   ... 
python -m scripts.evaluate     --config configs/example_m2.yaml

Branch selector

D8 is selected by measured affordability, not hope

N

Full deployable tax

The matched-attacker H6-D matrix fits the calendar, with slack for benign N and seeds. H6-D can be the headline.

N-

Scoped deployable tax

A reduced matrix fits: one deployable route, fewer targets/seeds, and 5% FPR primary. Report scoped H6-D plus the oracle gap honestly.

F

Oracle frontier only

Even the reduced deployable matrix does not fit. Use the H6-A oracle frontier as the headline, and mark H6-D as not run / unresolved.

GO / NO-GO

H1 gate: all four must hold

1. Benign reproduced

Published-like LIBERO benign success on pinned seeds.

2. Targeted redirect

Attack reaches target action region over persistence window.

3. Coarse-goal signal

Metric (A) separation survives the operator-goal rung.

4. Branch selected

N/N-/F chosen from measured micro-bench values.

GO

Proceed to M2 calibrated floor detector, then M3 H6-A, then M4 H6-D if branch permits.

NO-GO

Record exactly which gate failed and why. Do not hide negative results; revisit D2 framing, env, or checkpoint before rerun.

Run protocol

Every run log gets the same protocol block

Paste into each run logwrite-once results
run_id:        <UTC timestamp>-<short-slug>
git_commit:    <hash>
hardware:      <A100|H100 — card+count> + precision bf16 + CUDA/driver/torch
config:        <path to pinned config>
seed(s):       <pinned, recorded>
hypothesis:    <H#>
expected:      <pre-registered prediction>
command:       <exact command>
results_path:  results/<timestamp>/...
observed:      <fill after run>
decision:      <what it changes>
one_variable:  <single variable changed vs comparison run>
  • seeds pinned and recorded.
  • environment captured, including CUDA/driver/torch and module strings.
  • checkpoint/data provenance has source, hash, date, licence.
  • outputs are write-once; no overwrite.
  • exactly one variable changed vs comparison run.
  • figures regenerate from logged data.
  • adversarial artifacts are quarantined.
  • no dataset, checkpoint, secret, PII, or third-party binary is staged.
