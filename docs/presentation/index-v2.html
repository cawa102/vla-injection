The model normally maps sight + instruction to an action. Slip in extra hidden text and the model produces the attacker's action.

Why it matters
  • The output isn't a label — it's a physical motion. A hijacked output is a wrong, possibly unsafe, act.
  • Inputs are just pixels and text, so they are attackable — an instruction can be hidden in the scene or the prompt.
  • RoboGCG (2025) injects adversarial text → near-arbitrary control of the action.
Why it's open
  • The attack's own authors show borrowed defenses fail, or destroy benign performance → a working defense is open.
  • The closest consistency monitor (actalign) is benign-only — no attacker, no false-positive control.
The question

Can an instruction-injection attack be caught by checking whether the robot's action stays consistent with a trusted task goal — while keeping false alarms rare and calibrated?

A Trusted-Goal
Consistency Detector

The idea

Watch the robot's action and compare it against the operator's trusted goal. If the action diverges beyond a calibrated threshold, flag it and halt the robot.

Trusted goal = the real task, set once by the operator through a channel the injected text can't reach — so it stays a fixed reference. The action is checked against that, not against the (attackable) instruction the model reads.

01 Attacker-aware — it will be evaluated against a real adversarial attack (RoboGCG), not just benign or out-of-distribution data.
02 False-positive-calibrated — the threshold will be tuned to a chosen benign error budget (1% , 5%) on held-out runs, because a detector that cries wolf gets switched off.
03 Lightweight by design — no chain-of-thought, no heavy VLM judge; targets vanilla OpenVLA, in simulation (LIBERO).
